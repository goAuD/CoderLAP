# Cloud Computing

## Schneller visueller Überblick

| Merkmal | Bedeutung |
|---|---|
| on-demand | nach Bedarf nutzbar |
| broad network access | über das Netzwerk erreichbar |
| resource pooling | gemeinsamer Ressourcenpool |
| rapid elasticity | schnell skalierbar |
| measured service | messbar, häufig nutzungsbasiert |

## Was ist Cloud Computing?

Laut der NIST-Definition ist Cloud Computing ein Modell, das:

- nach Bedarf verfügbar ist
- über das Netzwerk genutzt werden kann
- auf gemeinsam genutzten Ressourcen aufbaut
- schnell erweitert oder reduziert werden kann
- typischerweise als messbarer Dienst funktioniert

Einfach gesagt:

- nicht unbedingt alles auf dem eigenen Rechner oder im eigenen Serverraum läuft
- sondern man die Infrastruktur eines Dienstleisters nutzt

## Warum ist das gut?

Typische Vorteile:

- schneller Start
- weniger eigene Hardware
- flexible Erweiterung
- oft einfacherer Betrieb

## Wofür kann es verwendet werden?

Zum Beispiel:

- Betrieb einer Webanwendung
- Speicherung von Backups
- Dateifreigabe
- Entwicklungs- und Testumgebung
- E-Mail und Office-Paket

## Worauf muss man achten?

Cloud ist kein Wundermittel.

Weiterhin wichtig:

- Datenschutz
- Berechtigungsverwaltung
- Kostenkontrolle
- Verfügbarkeit
- Vendor-Lock-in-Risiko

## Nicht verwechseln

| Begriff | Was bedeutet das? |
|---|---|
| Cloud Computing | allgemeines Modell |
| Public Cloud | öffentliches Cloud-Modell |
| SaaS | fertige Software als Dienst |
| eigener Server | nicht Cloud, nur weil er aus der Ferne erreichbar ist |

## Prüfungstaugliche Formulierung

> Cloud Computing ist ein Modell, bei dem Rechenressourcen, Speicherplatz, Plattformen oder komplette Anwendungen über das Netzwerk nach Bedarf in Anspruch genommen werden.  
> Hauptmerkmale sind die On-Demand-Nutzung, der Netzwerkzugriff, der gemeinsame Ressourcenpool, die schnelle Skalierbarkeit und der messbare Dienstcharakter.  
> Der Vorteil ist die Flexibilität, aber man muss auch auf Sicherheits-, Datenschutz- und Kostenaspekte achten.

## Häufige Prüfungsfehler

- Cloud einfach als "Internet" definieren.
- Glauben, dass Cloud automatisch alles sicher macht.
- Die Skalierbarkeit nicht erwähnen.
- SaaS mit dem gesamten Cloud-Begriff verwechseln.
