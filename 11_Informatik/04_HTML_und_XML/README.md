# HTML und XML

## Gyors vizuális kép

| Nyelv | Mire való? | Példa |
|---|---|---|
| HTML | weboldal szerkezete és tartalma | címsor, bekezdés, link |
| XML | adatok strukturált tárolása / cseréje | konfiguráció, adatleírás, üzenet |

## Mi az a HTML?

A `HyperText Markup Language` a web alap jelölőnyelve.

Feladata:

- megmondani, mi a cím
- mi a bekezdés
- mi a lista
- mi a link
- mi a táblázat

Tehát:

- szerkezetet és jelentést ad a webes tartalomnak

## Mi az az XML?

Az `eXtensible Markup Language` egy általános jelölőnyelv, amely:

- adatok leírására használható
- saját tageket is lehet benne definiálni

Példa XML-szerű logikára:

- egy rendelés adatait vagy egy konfigurációt lehet strukturáltan leírni

## Mi a fő különbség?

### HTML

- előre definiált elemeket használ
- böngészők megjelenítésre értelmezik

### XML

- szabadabban kialakítható struktúrát használ
- nem a megjelenítés, hanem az adatleírás a fő célja

## Miért fontos?

Mert vizsgán gyakori kérdés:

- melyik mire való
- mikor melyiket használjuk

## Ma hol találkozunk velük?

### HTML

- minden klasszikus weboldalon

### XML

- konfigurációkban
- dokumentumformátumokban
- bizonyos webszolgáltatásokban
- szabványos adatcserében

## Ne keverd össze

| Fogalom | Mit jelent? |
|---|---|
| HTML | tartalom és dokumentumszerkezet a weben |
| XML | strukturált adatleírás |
| CSS | HTML megjelenésének formázása |
| JSON | másik adatcsere-formátum, nem markup nyelv |

## Vizsgán jól használható megfogalmazás

> A HTML és az XML egyaránt jelölőnyelvek, de eltérő céllal használjuk őket.  
> A HTML a weboldalak tartalmának és szerkezetének leírására szolgál, míg az XML strukturált adatok leírására és cseréjére használható.  
> A HTML előre definiált elemekkel dolgozik, az XML pedig rugalmasabban alakítható adatszerkezetet tesz lehetővé.

## Gyakori vizsgahibák

- Azt mondani, hogy a HTML adatcsere-formátum.
- Az XML-t megjelenítő nyelvnek tekinteni.
- A CSS-t a HTML részeként kezelni.
- Nem megemlíteni, hogy XML-ben saját elemeket is lehet definiálni.
