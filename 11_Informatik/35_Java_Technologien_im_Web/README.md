# Java Technologien im Web

## Lényeg 30 másodpercben

A **Java webes technológiák** alatt főleg azokat a megoldásokat értjük, amelyekkel Java-alapon webalkalmazásokat és webes backendeket készítünk.

Fontos példák:

- `Servlet`
- `JSP` történetileg
- `Spring` / `Spring Boot`
- `Jakarta EE`

## Gyors vizuális kép

| Technológia | Szerep |
|---|---|
| Servlet | HTTP-kérések kezelése |
| JSP | szerveroldali nézetkészítés, inkább történeti jelentőség |
| Spring Boot | modern Java backend és webapp fejlesztés |
| Jakarta EE | enterprise Java platform |

## Mire használják a Javát a weben?

- backend API-k készítésére
- üzleti logika megvalósítására
- nagyobb vállalati rendszerekhez
- skálázható szerveroldali alkalmazásokhoz

## Történeti és modern rétegek

### Régebbi alapok

- `Servlet`
- `JSP`

Ezek fontosak megérteni, mert sok mai megoldás ezekre is épül történetileg.

### Mai tipikus irányok

- `Spring Boot`
- `Jakarta EE`
- REST API-k
- microservice alapú backendek

## Miért erős a Java a weben?

- stabil, érett ökoszisztéma
- sok vállalati környezetben bevált
- jó framework támogatás
- erős tooling és nagy közösség

## Java és JavaScript: ne keverd össze

| Fogalom | Jelentés |
|---|---|
| Java | általános programozási nyelv, erős backend jelenléttel |
| JavaScript | elsősorban webes scripting nyelv, frontend és ma sok backend is |

## Mire kell figyelni?

- A Java webes szerepe főleg backend oldalon erős.
- A `JSP` ma már inkább történetileg fontos, nem ez a modern fő irány.
- Vizsgán érdemes külön beszélni a régebbi és a mai technológiákról.

## Vizsgán jól használható megfogalmazás

> A Java webes technológiái elsősorban szerveroldali alkalmazások, backend szolgáltatások és vállalati webes rendszerek fejlesztésére szolgálnak.  
> A klasszikus alapok közé tartozik a Servlet és a JSP, míg a modern gyakorlatban a Spring Boot és a Jakarta EE a legfontosabb irányok közé tartoznak.  
> A Java különösen erős stabilitás, skálázhatóság és enterprise használat terén.

## Gyakori vizsgahibák

- A Java-t összekeverni a JavaScripttel.
- A JSP-t mai főtechnológiának beállítani.
- Nem megemlíteni a backend szerepet.
- Nem hozni modern példát, például Spring Bootot.
