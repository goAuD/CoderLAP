# User Story und Story Board

## Schneller visueller Überblick

| Werkzeug | Wozu ist es gut? |
|---|---|
| User Story | kurze Formulierung einer Benutzeranforderung |
| Story Board | visuelle Darstellung des Ablaufs oder Nutzungsszenarios |

## Was ist eine User Story?

Es ist eine kurze, benutzerzentrierte Beschreibung.

Klassisches Muster:

```text
Als [Benutzer],
möchte ich [Funktion],
damit [Nutzen / Ziel].
```

Beispiel:

```text
Als Käufer möchte ich meine Favoriten speichern,
damit ich sie später schnell wiederfinde.
```

## Warum ist die User Story gut?

- fokussiert auf den Benutzer
- kurz und verständlich
- hilft bei der Priorisierung
- guter Ausgangspunkt für Gespräche

## Was ist ein Story Board?

Ein Story Board ist eine **visuelle Geschichtsskizze**.

Es zeigt:

- wie der Benutzer das System verwendet
- welche Schritte aufeinander folgen
- wo es Probleme oder Entscheidungspunkte geben kann

Es kann sein:

- eine Folge von Bildkacheln
- eine einfache Skizze
- eine Prozessbildreihe
- eine Darstellung von Benutzerszenarien

## User Story und Story Board: nicht verwechseln

| Begriff | Art |
|---|---|
| User Story | kurze textuelle Anforderung |
| Story Board | visuelle Darstellung |

## Worauf muss man achten?

- die User Story sollte nicht zu technisch sein
- das Story Board sollte nicht nur hübsch, sondern auch verständlich sein
- keines von beiden ersetzt vollständig eine detaillierte Spezifikation

## Prüfungstaugliche Formulierung

> Eine User Story ist eine kurze, aus der Benutzerperspektive formulierte Anforderung, die beschreibt, wer was möchte und zu welchem Zweck.  
> Ein Story Board hingegen stellt den Benutzerablauf oder ein Nutzungsszenario visuell dar.  
> Zusammen unterstützen sie das Verständnis der Anforderungen, die Kommunikation und die Planung.

## Häufige Prüfungsfehler

- Die User Story als technische Aufgabenbeschreibung behandeln.
- Das Story Board mit Programmcode oder einem Wireframe verwechseln.
- Die Benutzerperspektive nicht erwähnen.
- Behaupten, dass diese allein vollständige Spezifikationen sind.

## Schnelle Selbstkontrolle

1. Was ist das Wesentliche an einer User Story?
2. Wozu ist ein Story Board gut?
3. Wie lautet das klassische User-Story-Muster?
4. Was ist der Hauptunterschied zwischen beiden?
5. Warum unterstützen sie die Kommunikation?

## Kurzantworten zur Selbstkontrolle

1. Kurze, benutzerzentrierte Anforderungsbeschreibung
2. Es stellt den Ablauf oder das Szenario visuell dar
3. Als..., möchte ich..., damit...
4. Das eine ist textuell, das andere visuell
5. Weil sie ein gemeinsames Verständnis schaffen
