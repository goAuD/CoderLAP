# Code of Conduct

## Our Standard

CoderLAP is a practical study project used by students and teachers.

Participation should stay:

- respectful
- factual
- constructive
- focused on helping people learn

Unacceptable behavior includes:

- harassment or personal attacks
- insulting, humiliating, or dismissive language
- discriminatory or hateful behavior
- sharing private credentials, personal data, or server details without
  permission
- derailing contribution or review discussions into personal conflict

## Expected Collaboration Style

When contributing or discussing changes:

- criticize ideas, not people
- keep technical disagreements concrete and evidence-based
- prefer precise bug reports and reproducible examples
- respect that this repository is maintained with a school-focused scope
- do not pressure maintainers to open access, remove protections, or accept
  scope creep

## Scope

This code of conduct applies to:

- GitHub issues
- pull requests
- discussions, if enabled later
- review comments
- other project communication channels used for CoderLAP collaboration

## Enforcement

The maintainer may remove, edit, or reject contributions and comments that do
not follow this code of conduct.

Serious or repeated violations may lead to:

- comment deletion
- issue or pull request closure
- access restriction from project collaboration channels

## Reporting

For conduct issues or sensitive concerns, contact the maintainer privately
instead of escalating in a public thread.
